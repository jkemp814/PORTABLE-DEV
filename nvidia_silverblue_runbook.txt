NVIDIA on Fedora Silverblue - Command Runbook
Date: 2026-05-03

1) Diagnostics we ran

rpm-ostree status
rpm -qa | grep nvidia
lsmod | grep nvidia
lsmod | grep nouveau
ls /usr/lib/modules/$(uname -r)/extra/nvidia/
sudo akmods --force --kernels $(uname -r)
mokutil --sb-state

2) Normalize kernel args (remove old/mixed entries, keep clean set)

sudo rpm-ostree kargs \
  --delete-if-present=rd.driver.blacklist=nouveau,nova_core \
  --delete-if-present=modprobe.blacklist=nouveau,nova_core \
  --delete-if-present=rd.driver.blacklist=nouveau \
  --delete-if-present=modprobe.blacklist=nouveau \
  --delete-if-present=nvidia-drm.modeset=1 \
  --append=rd.driver.blacklist=nouveau \
  --append=modprobe.blacklist=nouveau \
  --append=nvidia-drm.modeset=1

3) Dracut config to omit Nouveau and include NVIDIA modules

sudo mkdir -p /etc/dracut.conf.d
printf '%s\n' \
'omit_drivers+=" nouveau "' \
'add_drivers+=" nvidia nvidia_modeset nvidia_uvm nvidia_drm "' \
| sudo tee /etc/dracut.conf.d/10-nvidia.conf

4) Rebuild initramfs and reboot

sudo rpm-ostree initramfs --enable
systemctl reboot

5) Post-reboot verification

cat /proc/cmdline
lsmod | egrep 'nouveau|nvidia'
nvidia-smi
journalctl -k -b | egrep -i 'nouveau|nvidia|NVRM|drm'
sudo tail -n 120 /var/cache/akmods/nvidia/595.71.05-1-for-6.19.14-300.fc44.x86_64.failed.log

6) Optional install command (only if NVIDIA packages were missing)

rpm-ostree install akmod-nvidia xorg-x11-drv-nvidia
